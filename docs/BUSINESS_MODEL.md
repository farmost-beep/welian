# Welian.app 商业模型规划

> **一句话商业模式：批发大模型 tokens 赚价差，Pro 会员卖洞察，免费记录养成习惯。**
> 本文所有财务数字为规划推演，采用"保守/基准/乐观"三情景，PMF 验证前不做任何绝对承诺。

> **v1.1 变更摘要**（相对 v1.0.0）：
> 1. 商业模式从"联点按动作计费"升级为"批发赚价差 + 洞察收费"双轮模式（§1.3）
> 2. 联点定价从"按动作固定扣费"改为"按实际 token 用量计费"（§2.2）
> 3. 新增 §1.3 方案C 架构说明：云端计费网关 + 开源自托管双模式
> 4. 新增 §4.4 批发价差经济模型：毛利结构拆解
> 5. 定价调整：Pro 会员 ¥29/月（年付 ¥299），含 500 联点/月
> 6. 护城河新增"便利性锁定"（§9.1）

## §1. 为什么是 Tokens 经济

### 1.1 四个理由

| # | 理由 | 说明 |
|:-:|:----|:----|
| 1 | **成本可变性** | AI 产品的边际成本不为零：每次 advise/拟稿都有真实 LLM 调用成本。纯订阅制下重度用户吃掉毛利，轻度用户感觉亏（流失）——按量计费让价格与成本对齐 |
| 2 | **价值对齐** | 用户每次消耗联点都对应一次明确价值交付（一份草稿/一次建议/一份周报），付费感知清晰，不为"没用到的功能"买单 |
| 3 | **中国充值心智** | 从游戏点卡到 Q币 到直播充值，中国用户对"充值-消耗"模式的接受度全球最高，教育成本为零 |
| 4 | **模型分级空间** | 国产模型与旗舰模型成本差 10-30 倍。tokens 经济天然支持"标准/增强/最强"模型分级定价，纯订阅制做不到 |

### 1.2 混合模式：订阅含额度 + 加油包

纯 PAYG（按量付费）没有收入基线、无习惯承诺；纯订阅无法覆盖用量长尾。
采用 **订阅（含月度联点额度）+ 加油包（超额补充）** 的混合模式——这也是
Cursor/Claude/GitHub Copilot 等 AI 产品收敛出的行业标准形态。

### 1.3 方案C：批发赚价差 + 洞察收费（v1.1 核心升级）

v1.0 的联点经济有一个结构性问题：如果用户自托管 local agent 并自带 API key，
联点计费形同虚设——Welian 云被绕过，收入基础不存在。

**方案C 解决了这个矛盾**，通过云端计费网关实现"批发采购 → 零售赚价差"：

```
商业用户：端侧 → Welian 云（计费网关）→ LLM Provider
                    ↑ 验证 token + 扣联点 + 批发 API key 转发
                    ↑ 价差 = 持续性收入

开源用户：端侧 → LLM Provider（自带 key，不计费）
                    ↑ 开源版免费，是信任背书和获客漏斗
```

**为什么这个模式在中国市场特别成立：**

| # | 理由 | 说明 |
|:-:|:----|:----|
| 1 | **便利性溢价真实存在** | 中国用户用 Claude/OpenAI 需海外手机号、外币信用卡、汇率波动——Welian 批发后用人民币买联点，价差来自便利性而非信息不对称 |
| 2 | **批发折扣是真实的** | MiniMax/DeepSeek/GLM 对大客户有阶梯折扣（月消费 5万+ 8折，20万+ 6-7折），聚合多用户流量达到批发门槛 |
| 3 | **与 SPEC §7.1 架构完全吻合** | cloud-worker 不再空转——它就是收费站；tokens.py 不再形同虚设——每次调用真实扣费 |
| 4 | **双模式兼容** | 商业版走云端计费，开源版保留直连（`--self-hosted`），不破坏开源承诺 |

**收入双轮：**

```
轮1 — 联点价差（高频·持续性）
  · 批发采购 LLM tokens，零售卖给用户
  · 毛利 60-70%（批发折扣 + 便利性溢价）
  · 制造付费习惯和依赖路径

轮2 — Pro 会员（低频·厚利·锁定）
  · ¥29/月 或 ¥299/年
  · 包含 500 联点/月 + 仪表盘 + 年报 + 组合分析
  · 毛利 >90%（功能已开发，边际成本≈0）
  · 真正赚钱的层
```

**关键洞察**：联点不是用来赚大钱的，是用来**制造付费习惯和依赖路径**的。
真正赚钱的是 Pro 会员——但用户不会直接买会员，得先通过联点体验到 AI 价值。

## §2. 联点体系设计

### 2.1 定义

> **1 联点 ≈ 1K tokens 的标准模型调用量**（含输入+输出，向用户隐藏技术细节，只呈现动作价格）

### 2.2 按 token 用量计费（v1.1 升级）

v1.0 采用"按动作固定扣费"，v1.1 改为**按实际 LLM token 用量计费**——因为 Welian 批发采购 LLM tokens 后零售给用户，收入与成本直接对齐。

**定价规则：**

| 项 | 联点 | 说明 |
|:----|:--:|:----|
| 1K input tokens | 1 点 | 输入便宜，用户发的消息 |
| 1K output tokens | 2 点 | 输出贵，LLM 生成成本高 |

**典型动作的实际消耗（估算）：**

| 动作 | 估算 tokens | 联点 | 对应场景 |
|:----|:--:|:--:|:----|
| AI 记录增强 | ~2K in + 0.5K out | 3 | 每次"记一下"后自动增强 |
| AI 拟稿 | ~3K in + 0.5K out | 4 | 一条消息草稿 |
| 见面功课 | ~4K in + 1K out | 6 | 会面前速览 |
| 建议引擎 | ~6K in + 1.5K out | 9 | 该联系谁+为什么+聊什么 |
| 社交周报 | ~8K in + 2K out | 12 | 每周推送 |
| 角色仪表盘 | ~5K in + 2K out | 9 | 每月角色回顾 |
| 年度关系报告 | ~15K in + 5K out | 25 | 每年一次，可分享 |

> 注：以上为估算值，实际按 cloud-worker 返回的真实 usage 计费。透明消耗：每次调用前显示预计消耗，账单可查每一笔。

### 2.3 模型分级倍率

| 档位 | 模型类别 | 倍率 | 适用 |
|:----|:----|:--:|:----|
| 标准 | 已备案国产模型（DeepSeek/Qwen/GLM） | ×1 | 日常全部场景，默认 |
| 增强 | 中档旗舰（Sonnet 级） | ×3 | 重要拟稿、复杂建议 |
| 最强 | 顶级旗舰（Opus/GPT-5 级） | ×10 | 年度报告、关键场合文稿 |

### 2.4 联点规则

- 订阅赠点：当月有效 + 可滚存 1 个月（防浪费焦虑，又保留续订动力）
- 加油包联点：**永不过期**（充值信任的底线）
- 可赠予：联点可转赠好友（社交产品的社交增长钩子）
- 透明消耗：每次调用前显示预计消耗，账单可查每一笔

## §3. 定价结构

### 3.1 价格表

| 档位 | 价格 | 含联点 | 权益 |
|:----|:----|:----|:----|
| **免费版** | ¥0 | 100 联点/月 | 记录层全部功能永久免费 + 尝鲜额度（约 10 次拟稿或 1 份周报+若干动作） |
| **Pro 会员** | ¥29/月（年付 ¥299） | 500 联点/月 | +角色仪表盘、年度报告、组合分析、增强模型不加倍率 |
| **加油包** | ¥10 / ¥50 | 100 / 600 | 永不过期，全档位可购 |

### 3.2 定价逻辑

- **¥29 锚点**：低于中国订阅心理阈值（视频会员 ¥25-30），决策无摩擦
- **免费版 100 点**：刚好体验完整价值闭环（记录→周报→建议→行动）一轮，但不够常态使用——转化钩子
- **Pro 的本质**：不是"更多点"，是"洞察层功能"——仪表盘/年报/组合分析是边际成本≈0的高毛利功能
- **加油包 ¥10/100 点**：入门体验，让用户习惯"充值-消耗"心智
- 记录层永久免费是战略而非慈善：**记录数据是建议质量的燃料，是转化率的地基**（数据飞轮）

## §4. 单位经济模型

### 4.1 成本基础（2026 国产模型行情）

| 项 | 数值 |
|:----|:----|
| 标准模型零售价 | 输入 ~¥2/M tokens，输出 ~¥8/M tokens |
| 标准模型批发价（月消费 5万+） | 输入 ~¥1.2/M tokens，输出 ~¥5/M tokens（8折） |
| 标准模型批发价（月消费 20万+） | 输入 ~¥0.8/M tokens，输出 ~¥3.5/M tokens（6-7折） |
| 单次 advise 实际成本（批发） | ~8K 输入 + 2K 输出 ≈ **¥0.02-0.03** |
| 1 联点批发成本 | ≈ ¥0.003-0.005 |
| 1 联点售价 | ¥0.10（按加油包 ¥10/100 点计） |
| **联点毛利率** | **~60-70%**（批发折扣 + 便利性溢价） |

### 4.2 典型用户月度经济

| 用户类型 | 月消耗联点 | LLM 批发成本 | 月收入 | 毛利 |
|:----|:--:|:--:|:--:|:--:|
| 免费活跃用户 | 100 | ¥0.4 | ¥0 | -¥0.4（获客成本） |
| Pro 会员（轻） | 300 | ¥1.2 | ¥29 | ¥27.8（96%） |
| Pro 会员（重）+1 加油包 | 700 | ¥2.8 | ¥39 | ¥36.2（93%） |
| 加油包用户（非会员） | 200 | ¥0.8 | ¥10 | ¥9.2（92%） |

**关键结论**：
- 免费用户的服务成本 <¥0.5/月——免费层可以近乎无限量供给，获客成本结构极优
- Pro 会员毛利 >90%——联点成本只是零头，真正赚钱的是会员费本身（洞察层功能边际成本≈0）

### 4.3 LTV / CAC

| 指标 | 保守 | 基准 |
|:----|:--:|:--:|
| 付费用户月流失率 | 8% | 5% |
| 付费生命周期 | 12.5 月 | 20 月 |
| ARPPU | ¥24/月 | ¥32/月 |
| **LTV** | ¥300 | ¥640 |
| 目标 CAC（LTV/3） | <¥100 | <¥213 |

PLG + 内容获客的实际 CAC 预计 ¥10-40（见 §6），LTV/CAC 有 7-15 倍安全边际。

### 4.4 批发价差经济模型（v1.1 新增）

方案C 的收入结构拆解：

```
收入 = 联点价差 + Pro 会员费

联点价差（轮1）：
  用户付 ¥0.10/联点
  Welian 批发成本 ¥0.003-0.005/联点
  价差 = ¥0.095-0.097/联点
  毛利率 = 95-97%（按批发价计）

  但要扣除：
  - 云基础设施（cloud-worker 转发）：~¥0.001/联点
  - 支付通道费（0.6%）：~¥0.0006/联点
  实际毛利率 ≈ 60-70%

Pro 会员费（轮2）：
  用户付 ¥29/月
  包含 500 联点（批发成本 ¥1.5-2.5）
  洞察功能（仪表盘/年报/组合分析）边际成本 ≈ ¥0
  毛利 = ¥29 - ¥2 = ¥27
  毛利率 = 93%
```

**为什么联点毛利"只有"60-70%而会员毛利93%？**

联点是"通道业务"——赚的是转发价差，有真实变量成本（LLM API + 基础设施）。
会员是"价值层业务"——赚的是功能解锁，边际成本趋近于零。

**所以策略是**：联点定低价（让用户敢于用、习惯用），会员定合理价（让用户为洞察付费）。
联点是"入口"，会员是"利润池"。

**批发折扣的规模效应：**

| 月消费量 | 批发折扣 | 联点毛利 | 说明 |
|:----|:--:|:--:|:----|
| <¥1万 | 标准价 | 50-60% | 初期阶段 |
| ¥5万+ | 8折 | 65-75% | ~5000 活跃用户 |
| ¥20万+ | 6-7折 | 75-80% | ~2万活跃用户 |

规模越大，批发折扣越深，毛利越高——这是方案C 的规模效应飞轮。

## §6. 增长模型

### 6.1 PLG 主循环

```
微信里"记一下"（零摩擦免费）
   → 数据积累 → 周报/建议开始"懂你"（价值显现）
   → 免费额度不够用（自然付费墙）
   → 订阅转化 → 更多使用 → 数据更厚 → 建议更准（个人数据网络效应 = 留存护城河）
```

### 6.2 裂变钩子

| 钩子 | 机制 | 对标 |
|:----|:----|:----|
| **年度关系报告** | 每年 12 月生成可分享卡片（"今年你陪伴了 XX 人、兑现了 XX 个承诺"） | 网易云年度报告、支付宝账单 |
| **联点赠予** | 老用户赠新用户 100 联点，双方各得 | Dropbox 双边激励 |
| **拟稿水印（可关）** | 免费版草稿尾部"来自 Welian 的心意"，会员可关 | 邮件签名式自然曝光 |

### 6.3 冷启动（Phase 1 种子 1,000 用户）

1. **创始用户网络**：邮储系/USTC 校友/民建/职教社四大圈层本身就是 P1 画像密集区——产品的第一批用户就是产品服务的对象
2. **开源社区**：social-agent GitHub 用户是天然极客种子，转化为云版用户或传播者
3. **方法论内容**：双关系模型/目标锚定是原创方法论，以公众号/知乎/即刻长文输出（"你的微信好友该分成两种"天然是爆款选题）
4. **垂直社群**：保险/理财顾问社群定向渗透（关系=收入人群，付费意愿最高）

## §8. 成本结构：AI-First 运营

### 8.1 "一个人 + AI 军团"模式

Welian 本身就是用 AI 建造的（social-agent 全程由 AI 结对开发，201 个测试用例），运营也延续该模式：

| 职能 | 承担方 | 人力成本 |
|:----|:----|:--:|
| 产品/架构决策 | 创始人 | — |
| 开发/测试/部署 | AI Agent（Claude Code / Devin CLI） | ~¥3-5K/月工具成本 |
| 客服一线 | AI Bot（产品自身即客服入口） | 联点成本内 |
| 内容营销初稿 | AI 生成 + 创始人定稿 | — |
| 设计 | AI 生成 + 模板化 | — |

**Year 1 总成本预算 <¥15 万**（LLM 批发采购 + 云基础设施 + 微信生态费用 + AI 工具订阅 + 合规备案），
对应 Year 1 收入 ¥14 万——**第一年即接近盈亏平衡**，无融资依赖。这是 AI 时代独立产品的
结构性优势：毛利 85% + 人力成本趋近于零 = 极低的存活线。

### 8.2 成本占收入比（Year 3 基准）

| 项 | 占比 |
|:----|:--:|
| LLM 批发采购 | 10-12% |
| 云基础设施（Cloudflare/存储/推送） | 3% |
| 支付通道 + 微信生态 | 2% |
| 合规/财税/法务 | 2% |
| 营销（裂变激励为主） | 8-10% |
| **净利率空间** | **~65-70%** |

## §9. 护城河与风险

### 9.1 护城河（按强度排序）

| # | 护城河 | 强度 | 说明 |
|:-:|:----|:--:|:----|
| 1 | **个人数据网络效应** | ★★★★★ | 用户数据越厚建议越准，迁移=归零，留存护城河而非获客护城河 |
| 2 | **方法论品牌** | ★★★★ | 双关系模型/目标锚定/兑现追踪是原创理论，先发定义品类语言 |
| 3 | **微信生态深度** | ★★★★ | Bot+小程序+支付全链路，海外竞品结构性进不来 |
| 4 | **便利性锁定（方案C）** | ★★★ | 用户习惯人民币买联点后，回到自己搞 API key 的摩擦大——便利性溢价形成切换成本 |
| 5 | **端云分离信任架构** | ★★★ | 隐私架构可审计，大厂做关系数据反而有信任包袱 |
| 6 | 批发规模效应 | ★★ | 用户量越大批发折扣越深，毛利越高——后发者没有量就没有价差空间 |
| 7 | LLM 技术本身 | ★ | 无壁垒——所以壁垒必须建在数据、方法论和生态上 |

### 9.2 风险与对策

| 风险 | 概率 | 对策 |
|:----|:--:|:----|
| 微信官方做类似功能 | 中 | 微信是平台不做垂直参谋（历史一贯）；深耕方法论差异；多端布局（Web/CLI）保留退路 |
| LLM 价格战挤压联点心智（模型免费化） | 中 | 联点定价锚定"动作价值"而非"token 成本"；成本降=毛利升，及时下调价格让利即可 |
| 数据合规收紧 | 中 | 端云分离先天合规友好；国产已备案模型为默认通道；随时可切换纯端侧模式 |
| 转化率不及预期（<3%） | 中 | PMF 门槛制（§10），不达标不放大投入；调价/调免费额度 A/B |
| 社交异化舆论（"AI 教人功利社交"） | 低 | 双关系模型+伦理十不做就是公关答案；维系型关系故事优先传播 |
| 单点依赖创始人 | 高 | AI-first 运营降低运营依赖；开源引擎保证产品可延续 |
| 批发折扣不如预期（量不够） | 中 | 初期毛利 50-60% 也可维持；多 provider 比价；量起来后重谈折扣 |
| 用户绕过云端自托管（开源版） | 低 | 开源版是设计而非漏洞——自托管用户是极客少数，且成为传播者；商业用户要便利性不要 API key |

## §10. 里程碑与 PMF 验证门槛

> **原则：每道门槛未过，不进入下一阶段投入。数据先于扩张（继承 ROADMAP_v5 原则十三）。**
