# Welian 关系网络智能体产品规约

> **v3.0 核心跃迁：从"关系操作系统"到"关系网络智能体"。**
>
> v1 定义了双关系模型和四个核心场景（记/问/拟/报），验证了产品价值。
> v2 试图把 Welian 做成"关系的 iOS"——一个其他应用跑在上面的平台。
> v3 放弃平台幻想，回归本质：**Welian 是一个持续进化的智能体，
> 它越用越懂你的关系网络，主动帮你成为更好的社会角色。**
>
> 不是操作系统（用户不会在上面开发应用），不是工具（用完就走），
> 是一个**活的数字伙伴**——它记住、它理解、它主动行动、它在进化。

---

## §0. 为什么要从"操作系统"转向"智能体"

### 0.1 v2 的假设哪里错了

| v2 假设 | 现实 | 教训 |
|:----|:----|:----|
| 用户需要"关系平台"，第三方会在上面开发应用 | 没有开发者生态，用户也不关心"平台" | 平台隐喻对单人产品不成立 |
| 操作系统隐喻帮助用户理解产品 | 用户不理解"关系OS"，只关心"它能帮我做什么" | 隐喻太抽象，不驱动行为 |
| 核心价值是"让数据流向其他工具" | 用户要的是"帮我做事"，不是"帮我管数据" | 数据流通是手段不是目的 |

### 0.2 v3 的核心洞察

**用户不需要一个关系操作系统，用户需要一个关系智能体。**

| 操作系统 | 智能体 |
|:----|:----|
| 你给它指令，它执行 | 它理解你，主动行动 |
| 上面跑别人的应用 | 它本身就是应用 |
| 数据是核心资产 | 理解是核心资产 |
| 越用越依赖平台 | 越用越懂你 |
| 静态的（功能固定） | 进化的（持续学习） |

**关键转变**：从"Welian 帮你管关系数据"到"Welian 理解你的关系网络，并持续进化"。

### 0.3 "持续进化"的含义

Welian 的进化不是版本号递增，是**单个用户内、随使用时间增长的智能增长**：

| 进化维度 | 第1天 | 第30天 | 第90天 | 第一年 |
|:----|:----|:----|:----|:----|
| 关系理解 | 知道联系人名字 | 理解关系类型和亲疏 | 掌握互动模式和频率 | 预测关系走向 |
| 记忆深度 | 0 条互动 | ~30 条互动 | ~100 条互动 | ~400 条互动 |
| 主动建议质量 | 通用模板 | 基于个人画像 | 基于关系历史 | 基于网络全局 |
| 信号精准度 | 行业通用 | 匹配联系人领域 | 关联具体人和事 | 预判关系契机 |

## §1. 使命与定位

### 1.1 使命（不变）

> **让每个人成为更好的社会角色——更好的朋友、更好的家人、更好的合作者——最终成为更好的自己。**

### 1.2 定位（v3 重新定义）

| v1.0 | v2.0 | v3.0 |
|:----|:----|:----|
| 社交垂直应用 | 关系操作系统 | **关系网络智能体** |
| 帮你管好关系的工具 | 所有关系场景的智能中间层 | 持续进化的关系数字伙伴 |
| 用完就走 | 常驻后台 | **越用越懂你，主动行动** |

### 1.3 品牌（不变）

- **Welian = 维联**："维"系情感，"联"结目标
- **昵称：小维**
- **符号：∞**（无限进化）
- Slogan：**关系的质量，决定人生的质量**
- Eyebrow：**每段关系都值得用心**

### 1.4 一句话定位

> **Welian 是你的关系网络智能体——它记住你的每段关系，理解你的社交模式，主动提醒你该联系谁、该聊什么，并在持续进化中越来越懂你。**

| 动词 | 场景 | 示例 | 状态 |
|:--:|:----|:----|:----:|

## §5. 进化机制（v3 新增）

### 5.1 进化是什么

Welian 的进化是**单个用户内、随使用时间增长的智能增长**。不是产品版本迭代，是用户专属智能体的成长。

### 5.2 进化的四个引擎

| 引擎 | 机制 | 效果 | 状态 |
|:----|:----|:----|:----|
| **数据飞轮** | 每次对话自动提取联系人/互动/待办/记忆 | 数据越多，理解越深 | ✅ 已上线 |
| **画像自学习** | AI 从对话中自动更新用户画像 | 建议越来越个性化 | ✅ 已上线 |
| **目标追踪** | 设定关系目标，AI 自动从对话中匹配证据 | 目标兑现可追踪 | ✅ 已上线 |
| **技能学习** | 用户自定义技能，AI 按意图加载 | 智能体学会用户的方法论 | ✅ 已上线 |

### 5.3 进化里程碑

| 阶段 | 触发条件 | 智能体能力跃迁 |
|:----|:----|:----|
| **初生** | 注册完成 | 基础对话 + 通用建议 |
| **启蒙** | ≥3 联系人 + ≥1 互动 | 个性化建议 + 关系上下文 |
| **成长** | ≥10 联系人 + ≥20 互动 | 信号精准匹配 + 冷却预警 |
| **成熟** | ≥30 联系人 + ≥100 互动 | 关系模式识别 + 主动行动建议 |
| **精通** | ≥50 联系人 + ≥300 互动 | 网络全局视角 + 关系走向预判 |

### 5.4 进化的伦理边界

- 进化只让智能体更懂你，不让它替代你
- 陪伴型关系的"进化"只加深理解，不引入计量和评判
- 用户可以随时查看、编辑、删除智能体的记忆
- 进化的目的是帮用户用心，不是让 AI 替用户用心

## §11. 不做什么

**继承 v1 十不做：**
1. 不做社交平台
2. 不做聊天记录自动导入
3. 不对陪伴型关系做 ROI/排序/冷却
4. 不做幸福评分
5. 不做广告
6. 不做关系数据变现
7. 不做自动发消息/自动点赞
8. 不做无限建议
9. 不做订阅锁数据
10. 不做"附近的人"/LBS 社交

**v3 新增：**
11. **不做平台**——不做操作系统，不做应用商店，不做开发者生态。Welian 是智能体，不是平台。
12. **不做自动社交机器人**——AI 辅助但不替代人的用心。智能体建议，人决定。
13. **不做关系评分排名**——不对关系做"价值排名"暴露给用户。进化用于理解，不用于评判。

**Welian 的护城河**：
1. **双关系模型**——竞品都是单一模型，对家人做 ROI 是伦理硬伤
2. **智能体进化**——越用越懂你，竞品是静态工具
3. **多通道主动推送**——竞品都是"打开才有用"，Welian 在用户已有通道里主动出现
4. **信号引擎**——竞品只消费用户输入，Welian 主动从外部世界获取信息
5. **中国可用**——Clay/Dex 依赖 Gmail+LinkedIn，中国不可用
