# 自主循环

你要反复处理同一个任务直到完全完成。每次迭代你都能看到之前的工作成果（文件和 git 历史），在此基础上改进和推进。

## 任务

$ARGUMENTS

## 规则

1. **持续迭代**：完成一轮后不要停止，继续改进和推进直到任务完全完成
2. **基于前次成果**：每次迭代先检查之前做了什么（git log、git diff、文件状态），然后继续推进
3. **逐步推进**：每轮聚焦一个可验证的子目标，完成后继续下一个
4. **验证每步**：每轮结束时验证当前状态（运行测试、检查文件、确认功能）
5. **完成条件**：只有当任务完全且毫无疑问地完成时才停止。不要因为"差不多了"就退出
6. **遇到阻塞**：如果遇到只有用户能解决的问题，停下来说明情况

## 输出格式

每轮迭代结束时输出：
```
🔄 第 N 轮迭代完成
✅ 本轮完成：[具体完成的事]
📋 下一步：[下一轮要做的事]
📊 进度：[X/Y 子任务完成]
```

当任务完全完成时：
```
✅ 任务完成
📋 完成内容：[列表]
📊 总计 N 轮迭代
```
