## 代码审查

对当前 git 改动进行多维度审查。

### 步骤

1. 运行 `git diff HEAD` 获取所有改动
2. 运行 `git log --oneline -5` 了解最近提交上下文
3. 从以下维度并行审查：

**Bug 检测**：
- 语法错误、类型错误、缺失导入
- 逻辑错误（代码在所有输入下都会产生错误结果）
- 只看 diff 本身，不读额外上下文

**安全审查**：
- 命令注入风险
- XSS / CSRF 风险
- 硬编码密钥/密码
- SQL 注入
- 不安全的反序列化

**代码质量**：
- DRY 原则违反
- 错误处理缺失
- 边界条件遗漏
- 项目约定违反（检查 AGENTS.md / CLAUDE.md）

4. 只报告高置信度问题。不报告：
   - 代码风格偏好
   - 依赖具体输入的潜在问题
   - 主观建议
   - linter 能捕获的问题

5. 输出格式：
```
📋 代码审查结果

🔴 严重问题（必须修复）：
  · [文件:行号] 问题描述

🟡 重要问题（建议修复）：
  · [文件:行号] 问题描述

🟢 建议（可选）：
  · [文件:行号] 建议

✅ 没有问题的方面：
  · ...
```

如果没有发现问题，直接说"✅ 没有发现问题。已检查 bug、安全和代码质量。"
